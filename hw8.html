XQuery and XQIB and What do they do if applied on XML?

XQuery and its main characteristic

The short and the most precise way to characterize XQuery would be to say - XQuery is to XML what SQL is to databases. It is designed to query XML data and you can use XQuery to generate summary reports, transform XML data to HTML or search web documents for relevant information. It also could be used for pulling data from databases to be used for the application integration and splitting up an XML document that represents multiple transactions into multiple XML documents.Very powerful and convenient!

XQuery 3.1 became a W3C Recommendation on March 21, 2017 and XQuery 3.0 adds support for full functional programming, in that functions are values that can be manipulated (stored in variables, passed to higher-order functions, and dynamically called).

Learn by Example

XQuery What you will get in XML Explanations

xquery version "1.0";

for $b in
doc("hw8.xml")/bookstore/book

where
$b/price < 30

return $b/title

<title lang="en">Harry Potter</title>

<title lang="en">The Snowy Days</title>

xquery version "1.0" is a version declaration . The doc() function you see in the example shown
here, is used to open the hw8.xml file. The for clause binds a variable — the $b variable — to each item returned
by the in expression. The where clause is used to specify one or more criteria for the result as a filter. And in this example we will pull books which cost less then $30. Then, you
see a path expression that is used by XQuery to navigate through elements in an XML document — the path
expression /bookstore/book selects the book element using /bookstore followed by /book that selects all the book
elements that are children of the bookstore element.
And Return will define what to be returned. In our case return $b/title it will be the title of the book.

XQIB and how it can be used in XQuery

Although I could not find the exact meaning of what is XQIB, but it allows you to program client-side Web applications with XQuery instead of, or as a complement to JavaScript. It works in most modern browsers (IE9, Firefox 3.6+, Chrome 8+, Safari 5) and does not require the installation of any plugin.

XQuery is a W3C standard, just like HTML and XML, and is perfectly tailored for working with HTML, XML and REST. This means more elegant programs and less code. Moreover, by using an XQuery Web application server (such as MarkLogic or Sausalito) in conjunction with XQIB, having the same language on both the server and the client side allows you to reuse your code.
